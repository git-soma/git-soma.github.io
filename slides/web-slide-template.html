Opening

分享我如何把一个想法做成网页版 Slide

不用 PowerPoint，也不用复杂构建工具。一个 HTML 文件就能做出演示页、侧边栏导航、翻页动画、进度条和移动端自适应。

Static HTML CSS Grid Keyboard Navigation GitHub Pages

Part 1 / Structure

它本质上是一个静态网页

把每一页 slide 写成一个 HTML 区块，再用 CSS 控制布局，用 JS 切换当前页。

HTML

负责内容结构：封面、目录、卡片、表格、引用页。

CSS

负责视觉风格：纸张底纹、左侧导航、卡片阴影、排版比例。

JavaScript

负责交互：键盘翻页、按钮翻页、Hash 路由、进度条。

Part 2 / Workflow

制作时可以按 4 步走

01

先写大纲：每页只承载一个主要观点。

02

再定版式：封面、三栏卡片、双栏说明、资料表格、金句引用。

03

统一样式：颜色变量、字体、间距、边框和阴影。

04

最后加交互：导航、进度条、键盘快捷键和打印样式。

核心原则

它不是传统 PPT，而是一个“固定画布尺寸 + 自适应缩放”的网页。这样录屏、演讲、部署和分享都比较方便。

Part 3 / Components

常用页面组件

组件用途
Hero / Cover封面页，用大标题、导语和标签快速交代主题。
Card Grid把 3 个并列观点拆开，适合目录页、分类页、对比页。
Two Columns左侧讲流程，右侧放备注、案例或小结。
Material Table展示学习资料、链接、任务清单或工具对照。
Quote收尾页或转场页，用来制造记忆点。

Part 4 / Publish

部署到 GitHub Pages

把这个文件命名为 index.html，放到仓库中的某个目录。GitHub Pages 选择对应分支/目录后，就可以用浏览器访问。

/slides/my-talk/index.html GitHub Pages No build step

Closing

网页版 Slide 的优势，是把“内容、排版、交互、发布”都收束在一个网页里。
